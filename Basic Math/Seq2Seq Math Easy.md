## 1. 基本想法：為什麼需要 Seq2Seq？

有些任務的輸入和輸出，都是**一串有順序的資料**。

例如：

- 英文句子 → 中文句子
- 問題 → 回答
- 語音 → 文字

這種情況不能只輸入一個數字、輸出一個數字，而是要：

- 先讀完整個輸入序列
- 再一步一步生成輸出序列

這就是 **Seq2Seq（Sequence to Sequence）** 的基本想法。

最經典的做法是用兩個 RNN：

- **Encoder（編碼器）**：把輸入句子的資訊讀進去
- **Decoder（解碼器）**：把讀到的資訊轉成輸出句子

可以把它想成：

- Encoder 像是在「讀一句話並記住重點」
- Decoder 像是在「根據記住的重點，慢慢把另一句話說出來」

---

## 2. 基本符號定義

在推導之前，先把會用到的符號定清楚。

| 符號 | 意義 |
|------|------|
| $x_1, x_2, \dots, x_T$ | 輸入序列，第 $t$ 個輸入是 $x_t$ |
| $y_1, y_2, \dots, y_{T'}$ | 輸出序列，第 $t$ 個正確答案是 $y_t$ |
| $h_t$ | 第 $t$ 步的隱藏狀態（hidden state） |
| $\hat{y}_t$ | 第 $t$ 步模型預測的輸出 |
| $s_t$ | Decoder 在第 $t$ 步的狀態 |
| $c$ | context vector，Encoder 最後整理出的整句資訊 |
| $W_x$ | 輸入對狀態的權重 |
| $W_h$ | 前一個狀態對現在狀態的權重 |
| $b$ | 偏差（bias） |
| $\mathcal{L}$ | 損失函數 |

這裡故意不用太多矩陣，只把它看成：

- 現在的狀態 = 由「現在輸入」和「上一刻記憶」一起決定

## 3. 先理解單一步的 RNN

在一般神經網路裡，輸入一次就算一次。

但在 RNN 裡，**前一刻的資訊會傳到下一刻**。

### 3.1 一步的更新公式

RNN 在第 $t$ 步的核心公式是：

$$h_t = \tanh(W_x x_t + W_h h_{t-1} + b)$$

這句話的意思是：

- 看見現在的輸入 $x_t$
- 也參考上一刻的記憶 $h_{t-1}$
- 合起來得到新的記憶 $h_t$

### 3.2 直覺理解

可以把 $h_t$ 想成「讀到目前為止，腦中記得的重點」。

例如輸入句子是：

> I love cats

那麼：

- 讀到 `I` 時，得到 $h_1$
- 再讀到 `love` 時，得到 $h_2$
- 再讀到 `cats` 時，得到 $h_3$

所以 $h_3$ 會包含整句話到目前為止的大致意思。

### 3.3 為什麼常用 $\tanh$？

$\tanh$ 函數會把數值壓到 $-1$ 到 $1$ 之間：

$$\tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}}$$

它的好處是：

- 數值不容易爆掉
- 有正有負，比較適合表示「方向」或「傾向」

高中生只要先記得：

- 很大的正數進去，會接近 $1$
- 很小的負數進去，會接近 $-1$
- 接近 $0$ 的數進去，輸出也接近 $0$

## 4. Encoder：把整句話壓成一個狀態

Seq2Seq 的第一部分是 Encoder。

假設輸入序列長度是 $T$，那麼 Encoder 會做：

$$h_1 = \tanh(W_x x_1 + W_h h_0 + b)$$

$$h_2 = \tanh(W_x x_2 + W_h h_1 + b)$$

$$h_3 = \tanh(W_x x_3 + W_h h_2 + b)$$

一直到：

$$h_T = \tanh(W_x x_T + W_h h_{T-1} + b)$$

最後把

$$c = h_T$$

當成整個輸入句子的摘要，這個 $c$ 就叫做 **context vector**。

### 4.1 為什麼最後一個狀態可以代表整句？

因為每一步都把前面的記憶帶下來：

- $h_2$ 包含 $x_1, x_2$ 的資訊
- $h_3$ 包含 $x_1, x_2, x_3$ 的資訊
- ...
- $h_T$ 就包含整句輸入的資訊

所以在最基本的 Seq2Seq 中，我們直接用最後的 $h_T$ 作為整句摘要。

### 4.2 一句話版本

Encoder 的任務就是：

$$x_1, x_2, \dots, x_T \quad \longrightarrow \quad c$$

把一整串輸入，壓成一個向量狀態 $c$。

---

## 5. Decoder：一步一步產生輸出句子

有了 $c$ 之後，接著輪到 Decoder。

Decoder 的工作是：

- 先接收 $c$
- 然後從起始符號開始
- 每一步產生一個詞
- 再把這一步的資訊傳到下一步

### 5.1 Decoder 的起點

通常先把 Encoder 的摘要 $c$ 當成 Decoder 的初始狀態：

$$s_0 = c$$

其中 $s_t$ 表示 Decoder 在第 $t$ 步的狀態。

### 5.2 Decoder 的更新公式

Decoder 和 Encoder 很像，也是一個 RNN：

$$s_t = \tanh(W_y y_{t-1} + W_s s_{t-1} + b_s)$$

這裡要注意：

- $y_{t-1}$ 是前一步的輸出詞（或正確詞）
- $s_{t-1}$ 是前一步 Decoder 的記憶
- $s_t$ 是目前這一步要用來做預測的狀態

### 5.3 Teacher Forcing 的想法

在訓練時，常常不是把模型上一刻猜的字丟回去，而是把**正確答案的前一個字**丟回去。

例如要生成：

> 我 愛 你

則 Decoder 在訓練時可能會這樣看：

- 用 `<START>` 來預測「我」
- 用正確的「我」來預測「愛」
- 用正確的「愛」來預測「你」

這種方法叫 **Teacher Forcing**。

它的好處是訓練比較穩定，因為不會因為前一步猜錯而一路錯下去。

## 6. 輸出機率與預測詞

Decoder 在每一步算出狀態 $s_t$ 後，要把它變成「每個詞的機率」。

### 6.1 輸出層

先做一個線性變換：

$$o_t = W_o s_t + b_o$$

其中 $o_t$ 可以理解成：

- 對字典裡每個詞都打一個分數

例如字典只有三個詞：

- 我
- 愛
- 你

那麼某一步可能得到：

$$o_t = [2.0, 1.0, 0.1]$$

代表模型目前比較偏向第一個詞。

### 6.2 Softmax 變成機率

接著用 Softmax：

$$P(\hat{y}_t = k) = \frac{e^{o_{t,k}}}{\sum_j e^{o_{t,j}}}$$

這個公式的作用是把分數轉成機率，而且所有機率加起來會等於 $1$。

如果：

$$o_t = [2.0, 1.0, 0.1]$$

那麼轉成機率後，可能近似為：

$$[0.66, 0.24, 0.10]$$

這表示模型最可能選第一個詞。

### 6.3 預測規則

最後選機率最大的詞：

$$\hat{y}_t = \arg\max_k P(\hat{y}_t = k)$$

也就是：

- 哪個詞機率最大，就輸出哪個詞

## 7. 損失函數（Loss）

Seq2Seq 不是只預測一次，而是要預測很多步，所以損失也要一個時間點一個時間點算。

### 7.1 單一步的交叉熵

假設第 $t$ 步正確答案是某個詞 $y_t$，模型給這個詞的機率是 $P(y_t)$，那麼該步損失可以寫成：

$$\mathcal{L}_t = -\log P(y_t)$$

意思很簡單：

- 如果正確答案機率很高，損失很小
- 如果正確答案機率很低，損失很大

例如：

- 若 $P(y_t)=0.9$，則 $-\log(0.9)$ 很小
- 若 $P(y_t)=0.1$，則 $-\log(0.1)$ 很大

### 7.2 整句的總損失

如果輸出句子長度是 $T'$，總損失就是每一步加起來：

$$\mathcal{L} = \mathcal{L}_1 + \mathcal{L}_2 + \cdots + \mathcal{L}_{T'}$$

也就是：

$$\mathcal{L} = -\sum_{t=1}^{T'} \log P(y_t)$$

這就是最基本的 Seq2Seq 損失函數。

### 7.3 直覺理解

模型要學的事情就是：

- 在每一步，把正確答案的機率拉高
- 錯誤答案的機率壓低

只要整句每一步都做得越來越準，總損失就會下降。

## 8. 反向傳播的核心想法

真正訓練模型時，要讓參數（像 $W_x, W_h, W_o$）慢慢修正。

方法仍然是熟悉的：

- 先前向傳播得到預測
- 算出損失
- 再反向傳播算梯度
- 最後用梯度下降更新參數

### 8.1 先看輸出層

在第 $t$ 步，如果輸出層分數是 $o_t$，經過 Softmax 和交叉熵之後，有一個很重要的結果：

$$\frac{\partial \mathcal{L}_t}{\partial o_t} = \hat{p}_t - p_t$$

其中：

- $\hat{p}_t$ 是模型預測的機率分布
- $p_t$ 是正確答案對應的 one-hot 分布

這句話可以直覺理解成：

- 模型預測太高的地方，要往下修
- 模型預測太低的正確答案，要往上修

### 8.2 為什麼 RNN 比一般網路多一點難度？

因為第 $t$ 步的狀態 $h_t$ 會影響：

- 第 $t$ 步的輸出
- 第 $t+1$ 步的狀態
- 第 $t+2$ 步的狀態
- ...

也就是說，前面的狀態會影響後面很多步。

所以反向傳播時，誤差也要**沿著時間往回傳**，這就叫：

**BPTT（Backpropagation Through Time，時間上的反向傳播）**

### 8.3 不用太複雜的理解方式

把它想成骨牌：

- 第 1 步的狀態推動第 2 步
- 第 2 步推動第 3 步
- ...

所以如果第 5 步出錯，責任可能不只在第 5 步，也可能在第 4、3、2、1 步。

因此更新第 1 步相關參數時，要把後面所有影響都算進來。

### 8.4 梯度下降更新

更新規則仍然是：

$$W \leftarrow W - \alpha \frac{\partial \mathcal{L}}{\partial W}$$

其中：

- $W$ 代表任何一組權重
- $\alpha$ 是學習率

意思就是：

- 如果某個方向會讓損失變大，就往反方向走一點

## 9. 完整小範例：Hi → 嗨

下面用一個極簡化的例子，說明 Seq2Seq 在做什麼。

> 這裡是教學用示意，不追求真實語言模型的完整細節。

### 9.1 任務

輸入英文：

$$x_1 = \text{Hi}$$

輸出中文：

$$y_1 = \text{嗨}$$

我們假設輸出只有一個詞，所以 Decoder 只需要做一步。

### 9.2 Encoder

先把 `Hi` 經過 Encoder：

$$h_1 = \tanh(W_x x_1 + W_h h_0 + b)$$

因為只有一個輸入詞，所以最後：

$$c = h_1$$

### 9.3 Decoder 初始化

把 context 丟給 Decoder：

$$s_0 = c$$

接著輸入起始符號 `<START>`：

$$s_1 = \tanh(W_y \cdot <START> + W_s s_0 + b_s)$$

### 9.4 算輸出分數

假設字典裡只有三個候選詞：

- 嗨
- 你好
- 再見

Decoder 算出：

$$o_1 = [2.5, 1.2, 0.3]$$

### 9.5 轉成機率

用 Softmax 後，假設得到：

$$\hat{p}_1 = [0.70, 0.21, 0.09]$$

表示模型認為：

- 「嗨」機率 0.70
- 「你好」機率 0.21
- 「再見」機率 0.09

所以模型輸出：

$$\hat{y}_1 = \text{嗨}$$

### 9.6 損失

因為正確答案就是「嗨」，所以：

$$\mathcal{L} = -\log(0.70)$$

這個值不算太大，表示模型這次做得還可以。

如果模型把「嗨」的機率只算成 $0.05$，那損失就會大很多，表示模型做得很差。

## 10. 總結：Seq2Seq 的四個核心公式

如果只記最重要的內容，可以記下面四個式子。

### 10.1 Encoder 狀態更新

$$\boxed{h_t = \tanh(W_x x_t + W_h h_{t-1} + b)}$$

意思：

- 讀目前輸入
- 加上前一刻記憶
- 得到新的記憶

### 10.2 Context 向量

$$\boxed{c = h_T}$$

意思：

- 用 Encoder 最後一個狀態，當成整句摘要

### 10.3 Decoder 狀態更新

$$\boxed{s_t = \tanh(W_y y_{t-1} + W_s s_{t-1} + b_s)}$$

意思：

- 根據前一個輸出和前一刻記憶，決定現在狀態

### 10.4 整句損失

$$\boxed{\mathcal{L} = -\sum_{t=1}^{T'} \log P(y_t)}$$

意思：

- 每一步都希望正確答案機率高
- 全部步驟加總，就是整句的訓練目標

---

## 最後整理：一句話看懂 Seq2Seq

最基本的 Seq2Seq，就是：

1. 用 Encoder 把輸入序列一個一個讀完
2. 把最後的狀態當成整句摘要 $c$
3. 用 Decoder 以 $c$ 為起點，一步一步生成輸出
4. 用每一步的交叉熵加總成總損失
5. 用 BPTT 和梯度下降更新參數

所以它本質上就是：

$$\text{輸入序列} \rightarrow \text{句子摘要} \rightarrow \text{輸出序列}$$

這是後來很多翻譯模型、聊天模型、語音模型的重要起點。
